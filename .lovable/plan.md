

## Current Cancellation Policy

| Status | Cancel Button Visible? |
|--------|----------------------|
| `placed` | Yes |
| `accepted` | Yes |
| `preparing` | No |
| `ready` | No |
| All later stages | No |

The cutoff is when the seller starts **preparing** the order. This is similar to how Zomato/Swiggy work — once food preparation begins, cancellation is no longer allowed from the buyer side.

**Is this the behavior you want?** If you'd like to change the cancellation window (e.g., only allow cancel before acceptance, or add a refund policy for post-acceptance cancels), I can adjust the logic. No code changes needed for now — just confirming the current design.
